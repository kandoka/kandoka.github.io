【Java基础】重载与重写

发表于 2020-08-17 | 分类于 Java

重载（Overloading）

是什么

方法重载时让类以统一的方式处理不同类型数据的一种手段。多个同名函数同时存在，具有不同的参数和类型。重载是一个类中多态性的一种表现。

多态性体现在，调用方法时通过给这些重载方法传递不同的参数个数和参数类型来决定具体使用哪个方法，这就是多态性。

重载其实在Java中很常见，比如使用String.valueOf()时，根据里面传递的参数调用相应的方法，这样用户可以只用记住一个方法名将不同的数据类型转换成String类型。

重载的规则
  1. 必须具有不同的参数列表
  2. 可以由不同的返回类型
  3. 可以有不同的访问修饰符
  4. 可以跑出不同的异常

规则1是必须的才能实现重载，而规则2~4可选且不是必要的。

重写

是什么

重写一般发生在以下场景：

  1. 子类继承父类时，子类可以对父类的方法进行重写
  2. 子类继承抽象父类时，必须要重写父类的所有抽象方法
  3. 实现接口的时候要实现接口内的所有方法

这也是多态性的一种体现。

重写规则
  1. 参数列表必须与被重写方法相同
  2. 返回的类型必须一直与被重写方法的返回类型相同
  3. 访问修饰符的限制必须要大于等于被重写方法的访问修饰符，具体规则是（public > protected > default > private）
  4. 重写方法一定不能抛出新的检测异常，或者比被重写方法的异常类型更宽泛。

【Java基础】抽象类与接口

发表于 2020-08-17 | 分类于 Java

抽象类

是什么

首先抽象类是一种类，有自己的构造方法，只是它不能被实例化，只能被它的子类继承。抽象类的方法可以有普通方法和成员变量。而对于子类来说它只能继承一个抽象类，且子类必须要重写抽象类里的所有抽象方法。

接口

是什么

接口是与类并列的一个概念，没有自己的构造方法和成员变量， 它的变量都默认被public static final修饰变成了静态常量。接口中的方法默认都是public abstract的。对于普通类来说，可以同时实现多个接口， 但是要重写接口内的所有方法。

Reference
